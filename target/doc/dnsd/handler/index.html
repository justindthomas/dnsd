The DnsHandler trait is the boundary between the transport layer (UDP / TCP / DoT / DoH) and the query-processing layer (cache, forwarder, recursor, DNS64, DNSSEC).

Transports parse a single datagram / TCP message and call handle_bytes. The handler is expected to return a fully-formed DNS wire response, or None if the query was malformed beyond repair (in which case the transport silently drops — never reply to a malformed packet, that’s an amplification vector).

Using raw bytes instead of hickory_proto::op::Message lets every transport share the same dispatch without paying double encode/ decode; it also means DoH can pass POST bodies straight through.
